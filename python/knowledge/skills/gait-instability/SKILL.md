## 常见原因与检查项

### A. IMU 传感器问题

**检查项**:
- IMU 数据是否有异常噪声
- 零偏是否漂移
- 数据更新频率是否正常 (应 ≥ 200Hz)

**诊断命令**:
```bash
# 检查 IMU 数据
ros2 topic hz /rt/imu
ros2 topic echo /rt/imu --once
```

**解决**:
- 重新标定 IMU
- 检查线缆连接
- 必要时更换 IMU 模块

---

### B. 状态估计器漂移

**检查项**:
- EstimatorData 的 confidence 字段
- 位姿估计与实际情况是否一致
- 融合传感器数据是否正常

**解决**:
1. **重新初始化估计器**
   - 让机器人静止站立10秒
   - 重启 estimator 节点

2. **检查传感器融合**
   - 确认足力传感器数据正常
   - 检查关节角度反馈正常

3. **重新标定** (如上述无效)
   - 执行完整标定流程
   - 参考 sensor-calibration Skill

---

### C. 关节响应异常

**检查项**:
- 各关节跟踪误差 (position_tracking_error)
- 是否有关节力矩饱和
- 响应延迟是否 > 10ms

**常见故障关节**:
- 髋关节: 支撑力不足
- 踝关节: 姿态调整慢
- 膝关节: 屈伸不顺畅

**解决**:
- 调整伺服参数 (刚度、阻尼)
- 检查关节是否卡滞
- 参考 joint-overheat Skill 检查润滑

---

### D. 地面与环境因素

**检查项**:
- 地面是否平整
- 是否有湿滑区域
- 地面材质 (瓷砖/地毯/水泥)
- 光照是否影响视觉 (如有视觉定位)

**解决**:
- 避开湿滑/不平整区域
- 更换合适的鞋底材质
- 调整步态参数适应地面

---

## 处理方案

### 方案1: 紧急稳定 (已出现明显不稳)

**目标**: 立即稳定机器人姿态，防止摔倒

**步骤**:
1. **降低质心**
   - 命令机器人蹲下 (降低高度10-15cm)
   - 增大支撑多边形

2. **增大脚间距**
   - 调整为更宽的站姿
   - 提高横向稳定性

3. **减小刚度** (如震荡)
   - 降低关节伺服刚度
   - 增加阻尼

4. **原地待命**
   - 停止当前运动任务
   - 等待姿态稳定后再行动

---

### 方案2: 标定恢复 (估计器漂移)

**步骤**:
1. 让机器人在平坦地面静止站立
2. 执行 IMU 标定
3. 执行足力传感器标定
4. 重启状态估计器
5. 验证标定效果 (行走测试)

---

### 方案3: 参数优化 (特定地面)

**适用**: 特定地面类型上不稳定

**可调参数**:
```yaml
步态参数:
  step_height: 0.05      # 降低步高
  step_length: 0.3       # 减小步长
  stance_width: 0.25     # 增大支撑宽度
  
控制参数:
  kp_balance: 1.2        # 增大平衡控制增益
  kd_balance: 0.1        # 增大阻尼
  
接触参数:
  friction_coeff: 0.8    # 根据地面调整
```

## 相关案例

### CASE-2024-008: 瓷砖地面打滑
- **现象**: 行走时左右晃动，足力检测波动大
- **环境**: 光滑瓷砖地面
- **诊断**: 摩擦力不足，足底与地面接触不稳定
- **解决**: 更换橡胶鞋底，调整步态参数 (降低步高，增大支撑宽度)

### CASE-2024-019: IMU零偏漂移
- **现象**: 静止时姿态估计持续偏移，行走时偏向一侧
- **诊断**: IMU温漂导致零偏变化
- **解决**: 重新标定IMU，增加上电预热时间 (2分钟)

### CASE-2024-028: 多关节响应延迟
- **现象**: 行走时腿部动作不协调，有滞后感
- **诊断**: 通信延迟导致控制指令到达不同步
- **解决**: 检查网络，优化实时性配置

## 预防措施

1. **定期检查**
   - 每周执行姿态稳定性测试
   - 每月检查IMU零偏

2. **环境适配**
   - 新环境先进行适应性测试
   - 建立不同地面的参数配置文件

3. **监控指标**
   - 持续监控 balance_score
   - 设置 foot_force_variance 预警
